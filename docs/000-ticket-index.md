## 優先度

1. **最優先**: Phase 1-2（基本セットアップと認証）
2. **高**: Phase 3（コンテンツ表示）
3. **中**: Phase 4（進捗管理）、Phase 8（E2Eテスト）
4. **低**: Phase 5-7（管理画面とその後の改善）

## E2Eテストについて

Phase 8のE2EテストはPlaywright MCPを使用して実装します：
- **MCP (Model Context Protocol)** を通じてPlaywrightを操作
- ブラウザの自動操作とスナップショット取得
- CI/CDパイプラインでの自動テスト実行
- クロスブラウザ・クロスデバイステスト

## 開発の進め方

1. チケットを番号順に実装
2. 各チケットのTODOリストを確認
3. 実装完了後、チェックボックスを更新
4. PRを作成する際はチケット番号を含める（例: `feat: #001 データベース構築`）

## ファイル命名規則

- チケットファイルは `XXX-ticket-name.md` 形式
- XXXは001から始まる3桁の連番
- 例: `001-database-setup.md`, `072-auth-e2e-test.md`
